Learning Guide

Common techniques

This section is meant to be read like a compact playbook: each technique has a fast rule, a worked example, and a note on what the quiz is trying to train.

Logs and Exponentials

For numbers close to 1, write x = 1 + u and use ln(1 + u) ≈ u - u²/2. For base-2 logs, use log₂(x) = ln(x) / ln(2). For exponentials, use e^u ≈ 1 + u + u²/2 when u is small, and convert 2^u into e^{u \, ln(2)} when useful.

Examples: ln(1.08) ≈ 0.0768, log₂(10) ≈ 2.303 / 0.693 ≈ 3.32, and e^0.2 ≈ 1 + 0.2 + 0.02 = 1.22.
Why this works

The log and exponential cards are all different faces of the same local-linear idea. Near a familiar anchor, the Taylor series is doing the work: ln(1 + u) and e^u both have short truncations that stay accurate when u is small.

Base-2 logs are not a new object to memorize separately. The change-of-base identity turns them back into natural logs, and powers of 2 can be split into an easy integer part plus a small leftover fractional exponent.

Square Roots

Anchor at a nearby perfect square. If x = a + h and a is a square, use sqrt(a + h) ≈ sqrt(a) + h / (2sqrt(a)).

Example: sqrt(50) is near 49, so 7 + 1/14 ≈ 7.07.

Normal Tails

Memorize a few anchors and reuse them everywhere: 10% maps to about 1.28, 5% to 1.64, 1% to 2.33.

Example: if someone asks for P(Z > 1.64), your fast answer is about 5%.

Expected Max of Normals

For M_n = max(Z₁, ..., Z_n), start from P(M_n ≤ x) = Φ(x)^n. If the tail p = P(Z > x) is small, then Φ(x)^n = (1 - p)^n ≈ e^{-np}. Setting np ≈ 1 makes P(M_n ≤ x) ≈ e^{-1}, so this gives a central location for the max, not the exact mean. The true expectation is usually a bit above that anchor because the right tail still contributes.

Example: if n = 20, then 1 / n = 5%, so the shortcut anchor is x ≈ 1.64. That means the max is above 1.64 with probability about 1 - e^{-1} ≈ 63%, and the actual mean ends up a bit higher.
Why this works

The max is controlled by its upper tail. Since P(M_n > x) = 1 - Φ(x)^n, the place where Φ(x)^n stops being close to 1 is where the max starts to live. Writing Φ(x) = 1 - p with small p turns this into (1 - p)^n ≈ e^{-np}.

So np ≈ 1 identifies the scale where you expect about one exceedance among the n draws. That gives a location estimate, not the exact expectation, because the mean still averages over the whole right tail beyond that point.

Discrete Extremes and Full Sets

The same tail-threshold idea works for discrete variables. For maxima, solve n P(X ≥ k) ≈ 1. For minima, solve n P(X ≤ k) ≈ 1 or, when you have many category counts, center each count at its mean and use a left-tail normal anchor. Geometric tails are exact and Poisson counts are often well-approximated by normals with mean and variance λ.

Examples: for max of 16 geometric(1/2) variables, use 2^{-(k-1)} ≈ 1/16, so k ≈ 5. For 160 fair 20-sided dice, each face count has mean 8 and standard deviation about 2.76, so the expected number of removable full sets is roughly 8 - 1.64 × 2.76 ≈ 3.5.
Why this works

For maxima and minima, the first useful question is: where does the extreme stop being rare? That is exactly what the threshold rules n P(X ≥ k) ≈ 1 and n P(X ≤ k) ≈ 1 are measuring.

For the dice-set questions, the number of full sets is the minimum of the face counts. Those counts are dependent because they sum to the total number of dice, but after centering at the mean and scaling by the standard deviation, a normal extreme-value estimate still gives a good mental approximation for where the minimum usually lands.

Cards and Sampling

Deck and urn questions often reduce to indicator variables, symmetry, or simple without-replacement counting. If the question asks for an expected number of marked objects, try one indicator per object and add. If it conditions on part of the sample already seen, translate that condition into exact remaining counts.

Examples: expected fixed points in a 52-card shuffle is 52 × (1/52) = 1, and if 10 draws from a red-blue urn contain 6 red and 4 blue, the next-draw blue probability is just 6/10 from what remains.
What to look for

Indicators are especially strong when the objects are exchangeable. That is why fixed points, expected aces, color changes, and occupied faces all collapse quickly.

For order questions such as the first ace position, think of the special cards as a uniformly random subset of positions. Then symmetry or order-statistic formulas often replace any heavy combinatorics.

Random Walks on the Integers

For gambler’s ruin on {0, 1, ..., N}, the hit probability is x / N and the expected exit time is x(N - x). Those two formulas cover a lot of classic questions.

Example: start at x = 3 with barriers at 0 and 8. The expected exit time is 3 × 5 = 15.
Why this works

If h(i) is the probability of hitting N before 0 from state i, conditioning on the first step gives h(i) = (h(i-1) + h(i+1)) / 2 with boundary values h(0) = 0 and h(N) = 1. The only discrete-harmonic solution with those boundaries is linear, so h(i) = i / N.

For the expected exit time t(i), the same first-step conditioning gives t(i) = 1 + (t(i-1) + t(i+1)) / 2 with t(0) = t(N) = 0. That forces the second difference to be constant, so t(i) must be quadratic. Solving the boundary-value problem gives t(i) = i(N - i).

Rings and Regular Polygons

A walk on a regular polygon is a walk on a ring graph. Opposite-vertex hitting times reduce to distance times remaining distance, and geometry questions often reduce to familiar chord lengths.

Example: on a regular octagon, the opposite vertex is 4 edges away, so the expected hit time is 4(8 - 4) = 16.
Why d(n - d)?

A regular n-gon gives the cycle graph with n edges. For two vertices at graph distance d, the effective resistance on the cycle is d(n - d) / n, because the two arcs of lengths d and n - d act like two parallel resistors.

The commute-time identity says H(u,v) + H(v,u) = 2m R_eff(u,v), where m is the number of edges. On the cycle, m = n, so the commute time is 2d(n - d). By symmetry H(u,v) = H(v,u), so each one-way hitting time is d(n - d).

When the question asks for the probability of reaching the opposite vertex before coming back to a marked start vertex, symmetry folds the cycle into a gambler’s-ruin path. Starting one step away from the start on a path of length d gives probability 1 / d.

Markov Chain Spectra on Rings

These spectrum cards are about the transition matrix of the simple random walk Markov chain, not the adjacency matrix of the graph. On an n-vertex ring, one step averages the two neighbors, and that averaging rule produces eigenvalues cos(2πk/n).

Example: on an 8-vertex ring, the second-largest transition-matrix eigenvalue is cos(2π/8) = cos(π/4) ≈ 0.707, so the spectral gap is about 1 - 0.707 = 0.293.
Where the formula comes from

If P is the transition matrix, then (Pf)(j) = (f(j-1) + f(j+1)) / 2 because the walk moves left or right with probability 1/2. Now test the Fourier mode f_k(j) = e^{2πikj/n}.

Applying P gives (Pf_k)(j) = (e^{-2πik/n} + e^{2πik/n}) f_k(j) / 2 = cos(2πk/n) f_k(j). So each Fourier mode is an eigenvector, and its eigenvalue is exactly cos(2πk/n).

Moments and MGF

Second moments, variances, and mgfs are tightly connected. Use E[X²] = Var(X) + (E[X])² for quick second moments, add variances across independent pieces, and remember the hypergeometric correction for sampling without replacement. For mgf problems, either recognize the family or plug in a clever value of t.

Examples: if H ~ Binomial(10, 1/2), then E[H²] = 2.5 + 25 = 27.5. If X ~ Poisson(3), then M_X(ln 2) = exp(3(2 - 1)) = e³.
What to look for

Variance questions usually ask whether you are in the independent-sum world or the without-replacement world. The finite-population correction is the main mental difference between binomial variance and hypergeometric variance.

For mgfs, first ask whether the form matches a standard template. A normal mgf exp(μt + σ²t²/2) lets you read off mean and variance directly, while a finite weighted sum of e^{tx} often lets you read off the underlying point masses.

Stopping and Martingales

One-step stopping problems are threshold problems: compare what you can lock in right now to the continuation value of using the option to keep going. Martingale stopping problems are different: if the process is fair, optional stopping says timing alone does not create expected value.

Examples: with one free die reroll, the continuation value is 3.5, so keep 4, 5, 6. For a symmetric stock that moves up or down by 1 each minute, the expected sale price stays at the current price under any bounded stopping rule.
What to look for

If you get exactly one chance to continue, compute the continuation value first and keep only outcomes above it. That turns many interview stopping questions into simple compare-against-a-threshold drills.

If the underlying process is a martingale, separate the question of variance reduction from the question of expected value. Stopping can change the distribution, but under the standard fair bounded setup it does not change the mean.

Green Book: Noodles

If you have n noodles and randomly pair all loose ends, the expected number of loops is 1 + 1/3 + 1/5 + ... + 1/(2n - 1).

Example: for 4 noodles, the expectation is 1 + 1/3 + 1/5 + 1/7 ≈ 1.676.

Green Book: Runs

For n fair coin flips, the expected total number of runs is (n + 1) / 2. A good mental rule for the longest run of heads is that it sits around log₂ n.

Example: in 20 flips, the expected number of runs is 10.5. In 64 flips, the longest head run is mentally around 6.

Green Book: Waiting for a Streak

For any fair-coin pattern w, the expected waiting time until you first see it is sum 2^k over all lengths k for which the first k symbols of w equal the last k symbols of w. This is the standard martingale / betting-system result. Runs of heads are just the special case where every prefix is also a suffix, so HHH gives 2 + 4 + 8 = 14.

Examples: for HTH, the matching prefix/suffix lengths are 1 and 3, so the mean wait is 2 + 8 = 10. For HTHT, the matching lengths are 2 and 4, so the mean wait is 4 + 16 = 20.
Why overlap lengths appear

The clean proof starts a new bettor on every flip. Each bettor bets sequentially on the letters of the target pattern and doubles their stake whenever they survive one more step. With fair-coin payouts, the total wealth process is a martingale.

At the stopping time when the pattern first appears, exactly the bettors whose starting positions align with a prefix-suffix overlap are still alive. Their payouts are 2^k for the overlap lengths k, so optional stopping turns the expected waiting time into the sum of those powers of 2.

Green Book: Permutations and Coupons

Several classics live here. The expected number of cycles in a random permutation of n items is H_n, the expected number of fixed points is 1, the derangement probability is close to 1/e, and the expected time to collect n equally likely coupon types is n H_n.

Example: for 4 coupon types, the expected collection time is 4(1 + 1/2 + 1/3 + 1/4) ≈ 8.33.
