Learning Guide

Common techniques

Logs, Roots, and Exponents

Choose a nearby point where the function is easy to evaluate, then use one or two terms of a local expansion. Use ln(1+u) ≈ u - u²/2 when u is small, sqrt(a+h) ≈ sqrt(a) + h/(2sqrt(a)) when a is a nearby perfect square, e^u ≈ 1 + u + u²/2 when u is small, and log₂(x) = ln(x)/ln(2) for base-2 logs.

Examples: to estimate ln(1.08), write it as ln(1+0.08) and use 0.08 - 0.08²/2 = 0.0768. To estimate sqrt(50), anchor at 49 and use sqrt(49+1) ≈ 7 + 1/(2·7) = 7.07. To estimate log₂(10), compute ln(10)/ln(2) ≈ 2.303/0.693 ≈ 3.32.
Why this works

This is the Taylor-expansion idea. If f is smooth and h is small, then f(a+h) = f(a) + h f'(a) + O(h²). The first-order term gives the quick estimate; the second-order term tells you whether that estimate tends to be slightly high or slightly low.

For ln(1+u), the second-order correction is negative, so the linear estimate u overshoots. For e^u, the second-order correction is positive, so the linear estimate 1+u undershoots. For square roots, the derivative 1/(2sqrt(a)) explains why moving from 49 to 50 adds about 1/14, not about 1.

Mixed cards are not a separate theory. They just ask you to approximate each component with the correct local rule, then combine the results while keeping track of sign and scale.

Normal Tails

Throughout this deck, an “upper-tail anchor” means a statement of the form P(Z > z) ≈ α for Z ~ N(0,1). The anchors worth memorizing are explicit probability statements: P(Z > 1.28) ≈ 0.10, P(Z > 1.64) ≈ 0.05, P(Z > 1.96) ≈ 0.025, and P(Z > 2.33) ≈ 0.01. By symmetry, P(Z < -1.28) ≈ 0.10 and similarly for the others.

Example: if the prompt asks for P(Z > 1.64), use the memorized one-sided tail and answer about 0.05. If the prompt asks for P(|Z| > 1.64), double that and answer about 0.10. If the prompt asks for P(|Z| < 1.64), subtract both tails from 1 and answer about 0.90.
How to read these statements correctly

A statement like P(Z > 1.64) ≈ 0.05 is a one-sided upper-tail statement. It does not mean P(|Z| > 1.64); that two-sided probability is about 0.10. Likewise, P(Z < 1.64) is about 0.95, because it is the complement of the upper tail.

If the prompt gives a probability and asks for z, you are solving a quantile problem. For example, “find z such that P(Z > z) ≈ 0.05” means “find the number whose one-sided upper tail is about five percent,” which gives z ≈ 1.64.

This is why a short table is enough. Once you translate the question into upper tail, lower tail, or central probability, the rest is symmetry and complementation.

Expected Max of Normals

Let M_n = max(Z₁, ..., Z_n) for iid Z_i ~ N(0,1). The exact distribution is P(M_n ≤ x) = Φ(x)^n. The first mental anchor is the level u_n that solves P(Z > u_n) ≈ 1/n. This means that among n independent standard-normal draws, the expected number of draws above u_n is about 1. That level is a location estimate for the maximum, not the exact mean E[M_n].

Example: if n = 20, then 1/n = 0.05. The matching one-sided upper-tail normal level is about u_{20} ≈ 1.64, because P(Z > 1.64) ≈ 0.05. At that threshold, P(M_{20} ≤ 1.64) = (0.95)^{20} ≈ e^{-1} ≈ 0.37, so P(M_{20} > 1.64) ≈ 0.63. This shows what the anchor means: it is the point where the maximum has become quite likely to exceed the threshold, not a claim that the mean equals 1.64.
Why this works

Write p = P(Z > x). Then P(M_n ≤ x) = (1-p)^n. When p is small, (1-p)^n ≈ e^{-np}. So the level where np ≈ 1 is the level where P(M_n ≤ x) is about e^{-1}. That is the mathematical reason the “one expected exceedance” level is the correct first anchor.

This still does not equal the expectation. The quantity E[M_n] depends on the whole upper tail of the distribution of M_n, not just the single level where exceedances become order one. There is still substantial mass above u_n, so the actual expected maximum is somewhat larger.

For n = 2, there is an exact identity worth memorizing. Since max(Z₁,Z₂) = (Z₁ + Z₂ + |Z₁-Z₂|)/2 and E[Z₁+Z₂]=0, we get E[max(Z₁,Z₂)] = E|Z₁-Z₂|/2. Because Z₁-Z₂ ~ N(0,2), we have E|Z₁-Z₂| = sqrt(2)·sqrt(2/pi) = 2/sqrt(pi). Therefore E[max(Z₁,Z₂)] = 1/sqrt(pi) ≈ 0.5642, and by symmetry E[min(Z₁,Z₂)] = -1/sqrt(pi).

A sharper large-n refinement is E[M_n] ≈ u_n + γ a_n, where a_n ≈ 1/(n φ(u_n)) and γ ≈ 0.577. For a fast estimate, first find the 1/n upper-tail quantile; the expected maximum is slightly larger.

Discrete Extremes and Full Sets

The same tail-threshold idea works for discrete variables. For maxima, solve n P(X ≥ k) ≈ 1. For minima, solve n P(X ≤ k) ≈ 1 or, when you have many category counts, center each count at its mean and use a left-tail normal anchor. Geometric tails are exact and Poisson counts are often well-approximated by normals with mean and variance λ.

Examples: for max of 16 geometric(1/2) variables, use 2^{-(k-1)} ≈ 1/16, so k ≈ 5. For 160 fair 20-sided dice, each face count has mean 8 and standard deviation about 2.76, so the expected number of removable full sets is roughly 8 - 1.64 × 2.76 ≈ 3.5.
Why this works

For maxima and minima, the first useful question is: where does the extreme stop being rare? That is exactly what the threshold rules n P(X ≥ k) ≈ 1 and n P(X ≤ k) ≈ 1 are measuring.

For the dice-set questions, the number of full sets is the minimum of the face counts. Those counts are dependent because they sum to the total number of dice, but after centering at the mean and scaling by the standard deviation, a normal extreme-value estimate still gives a good mental approximation for where the minimum usually lands.

Cards and Sampling

Deck and urn questions often reduce to indicator variables, symmetry, or simple without-replacement counting. If the question asks for an expected number of marked objects, try one indicator per object and add. If it conditions on part of the sample already seen, translate that condition into exact remaining counts.

Examples: expected fixed points in a 52-card shuffle is 52 × (1/52) = 1, and if 10 draws from a red-blue urn contain 6 red and 4 blue, the next-draw blue probability is just 6/10 from what remains.
Exact workflow

Indicators are especially strong when the objects are exchangeable. That is why fixed points, expected aces, color changes, and occupied faces all collapse quickly.

For order questions such as the first ace position, think of the special cards as a uniformly random subset of positions. Then symmetry or order-statistic formulas often replace any heavy combinatorics.

Conditional Probability and Bayes Updates

Let h denote the hidden state, such as the selected coin, urn, or disease status. For each possible state, compute P(h) P(data | h), then divide by the sum over all states. Thus the posterior probability is proportional to prior probability times likelihood.

Examples: if one of three coins with head probabilities 1/4, 1/2, and 3/4 is chosen uniformly and you observe HH, then the posterior weights are proportional to (1/4)^2, (1/2)^2, and (3/4)^2, namely 1 : 4 : 9. So the probability the next toss is heads is the posterior weighted average (1/4)(1/14) + (1/2)(4/14) + (3/4)(9/14) = 9/14.
Exact workflow

Step 1: list the hidden states. Step 2: for each state, compute the likelihood of the observed data under that state. If tosses are conditionally iid given the chosen coin, the likelihood is just p^{#H}(1-p)^{#T}. If draws are without replacement from a chosen urn, the likelihood uses the changing remaining composition. Step 3: multiply prior by likelihood. Step 4: divide by the total to get posterior probabilities.

If the prompt then asks for a future event, do one more averaging step. For example, if a coin with head probability p_i has posterior weight w_i, then the posterior predictive probability of heads next is Σ w_i p_i. If the prompt instead asks which hidden state is most likely, stop after the posterior calculation.

Some “classic” conditional-probability questions are really sample-space questions rather than Bayes updates. The two-child puzzle is the standard warning sign: “the older child is a boy” leaves two equally likely families {BB, BG}, so P(BB)=1/2; “at least one child is a boy” leaves three equally likely families {BB, BG, GB}, so P(BB)=1/3. The conditioning statement matters because it changes which outcomes remain and how they are weighted.

Expected Value by Conditioning

When an expected-value question looks messy, first ask whether symmetry already fixes the answer. If not, condition on the first meaningful reveal, solve the smaller continuation problem, and average back up.

Examples: with one free reroll of a fair die, the continuation value is 3.5, so you keep 4, 5, 6. For the first ace in a shuffled deck, symmetry says the 4 ace positions form a random 4-subset of {1, ..., 52}, so the expected first ace position is (52 + 1)/(4 + 1) = 10.6.
Exact workflow

Linearity of expectation is the default tool when the quantity is a count. Symmetry is the default tool when every object has the same chance to be the special one. Conditioning is the default tool when the next reveal turns the problem into a smaller copy of itself.

That covers a lot of the database: cards, urns, coupon questions, stopping rules, and many “fair price” interview problems are all one of those three moves in disguise.

Random Walks on the Integers

For gambler’s ruin on {0, 1, ..., N}, the symmetric hit probability is x / N and the expected exit time is x(N - x). For a biased walk that moves right with probability p and left with probability q = 1-p, the upper-hit probability becomes (1 - (q/p)^x)/(1 - (q/p)^N).

Examples: start at x = 3 with barriers at 0 and 8. The expected exit time is 3 × 5 = 15. If the walk is biased with p = 0.6, q = 0.4, and starts at x = 2 on {0,1,…,5}, then the chance of hitting 5 before 0 is (1 - (2/3)^2)/(1 - (2/3)^5).
Why this works

If h(i) is the probability of hitting N before 0 from state i, conditioning on the first step gives h(i) = (h(i-1) + h(i+1)) / 2 with boundary values h(0) = 0 and h(N) = 1. The only discrete-harmonic solution with those boundaries is linear, so h(i) = i / N.

For the expected exit time t(i), the same first-step conditioning gives t(i) = 1 + (t(i-1) + t(i+1)) / 2 with t(0) = t(N) = 0. That forces the second difference to be constant, so t(i) must be quadratic. Solving the boundary-value problem gives t(i) = i(N - i).

In the biased case, the recurrence becomes h(i) = p h(i+1) + q h(i-1). The general solution is A + B(q/p)^i, and the boundary values h(0)=0, h(N)=1 force h(i) = (1 - (q/p)^i)/(1 - (q/p)^N). When p = q = 1/2, that formula limits back to i/N.

Hitting-Time Variance

For the same walk on {0, 1, ..., N}, the exit-time variance is x(N - x)(N² - 2 - 2x(N - x)) / 3. The mental shortcut is to compute a = x(N - x) first, then plug into a(N² - 2 - 2a)/3.

Example: if the walk starts at x = 3 and stops at 0 or 8, then a = 15, so Var(τ) = 15(64 - 2 - 30)/3 = 160.
Where the quartic martingale enters

The quadratic martingale X_t² - t is enough to get E[τ] = x(N - x). To get the second moment of the stopping time, you need one more layer: stop the quartic martingale X_t^4 - 6tX_t² + 3t² + 2t at the exit time τ.

At time τ, the walk sits at 0 or N, so the stopped identities involve only boundary values, E[τ], and E[τ²]. Solving those equations gives E[τ²] = x(N - x)(N² - 2 + x(N - x))/3. Subtracting E[τ]² then yields Var(τ) = x(N - x)(N² - 2 - 2x(N - x))/3.

Rings and Regular Polygons

A walk on a regular polygon is a walk on a ring graph. If the polygon has an even number of vertices and you start at vertex A, then “the vertex opposite A” means the unique vertex n/2 edges away from A. If a prompt says “a specific marked vertex B that is d edges away from A,” that means one designated target vertex, not either of the two symmetric choices.

Example: on a regular octagon, the unique vertex opposite A is 4 edges away, so the expected time to hit it from A is 4(8 - 4) = 16.
Why d(n - d)?

A regular n-gon gives the cycle graph with n edges. If A and B are at graph distance d, then the two arcs from A to B have lengths d and n - d. In the electrical-network picture, those are parallel resistors, so the effective resistance is d(n - d) / n.

The commute-time identity says H(A,B) + H(B,A) = 2m R_eff(A,B), where m is the number of edges. Here m = n, so the commute time is 2d(n - d). The cycle is symmetric under swapping A and B, so H(A,B) = H(B,A). Therefore each one-way hitting time is d(n - d).

For probability cards, keep the labels straight. Let A be the bad boundary and O the unique vertex opposite A on an even n-gon. If the walk starts at the specific vertex lying i edges from A along one chosen arc from A to O, folding the cycle turns the problem into gambler’s ruin on {0,1,…,d}, where d = n/2. The probability of hitting O before A is then exactly i/d. In particular, if you start at the specific neighbor of A on that arc, the probability is 1/d.

Markov Chain Spectra on Rings

These spectrum cards are about the transition matrix of the simple random walk Markov chain. On an n-vertex ring, one step averages the two neighbors, and that averaging rule produces eigenvalues cos(2πk/n).

Example: on an 8-vertex ring, the second-largest transition-matrix eigenvalue is cos(2π/8) = cos(π/4) ≈ 0.707, so the spectral gap is about 1 - 0.707 = 0.293.
Where the formula comes from

If P is the transition matrix, then (Pf)(j) = (f(j-1) + f(j+1)) / 2 because the walk moves left or right with probability 1/2. Now test the Fourier mode f_k(j) = e^{2πikj/n}.

Applying P gives (Pf_k)(j) = (e^{-2πik/n} + e^{2πik/n}) f_k(j) / 2 = cos(2πk/n) f_k(j). So each Fourier mode is an eigenvector, and its eigenvalue is exactly cos(2πk/n).

Sharpe from a PnL Graph

If a graph shows cumulative PnL values C_0, C_1, ..., C_m at equally spaced times, recover each period's PnL as R_i = C_i - C_{i-1}. The period Sharpe ratio is mean(R_i) / sd(R_i). For annualized Sharpe, multiply by sqrt(periods per year).

Example: if monthly cumulative PnL is 0, 2, 2, 4, 4, 6, then the monthly increments are 2, 0, 2, 0, 2. Their mean is (2+0+2+0+2)/5 = 1.2. Their standard deviation is about 0.98. So the monthly Sharpe is about 1.2/0.98 ≈ 1.22. If the prompt asks for annualized Sharpe, multiply by sqrt(12) to get about 4.2.
Exact workflow

Step 1: convert cumulative PnL into period increments. Step 2: compute or estimate the average increment. Step 3: compute or estimate the standard deviation of the increments. Step 4: divide mean by standard deviation. Step 5: annualize only if the prompt explicitly asks for it.

On an easier marked chart, you can read off enough increments to estimate them directly. On a dense chart, the intended judgment is visual: a steeper average slope means larger mean increment, and a rougher path around that trend means larger standard deviation. In that dense case, you are estimating the ratio “drift per period divided by wiggle per period,” not reconstructing every single increment exactly.

If the graph is plotted in dollars, percent of capital, or some other linear unit, the Sharpe is unchanged by rescaling the vertical axis, because both the numerator and denominator scale by the same factor.

Sharpe and Return Probabilities

If a return R has Sharpe ratio S and standard deviation σ for a given period, then its mean return for that same period is μ = Sσ. After that, there are two different kinds of questions. If the prompt gives a distributional model, such as approximate normality, compute a z-score and use that model. If the prompt gives no distribution, use a bound that depends only on mean and variance.

Example: if monthly Sharpe is 0.5 and monthly volatility is 2%, then the monthly mean is μ = 0.5 × 2% = 1%. If the prompt says returns are approximately normal, then P(R > 3%) = P((R-μ)/σ > (3%-1%)/2%) = P(Z > 1), which is about 0.16. If no distribution is given and the prompt asks for a bound on P(R < 0), then the relevant distance below the mean is a = μ, and Cantelli gives P(R-μ ≤ -a) ≤ σ²/(σ²+a²).
Which probability statement you are actually bounding

If the prompt asks for the probability of a negative return and the return has mean μ > 0, then you are bounding P(R < 0) = P(R-μ ≤ -μ). Cantelli therefore gives P(R < 0) ≤ σ²/(σ² + μ²). If you rewrite μ = Sσ, this becomes P(R < 0) ≤ 1/(1+S²). That is the clean mental formula when only Sharpe is given.

If the prompt instead asks about exceeding some target k, write the event as R-μ ≥ k-μ or R-μ ≤ k-μ and then choose the correct one-sided bound. Normality gives a sharper answer when it is allowed, because you can compute the actual z-score rather than only a worst-case bound.

Chebyshev is two-sided and therefore weaker when the question is one-sided. Markov applies only to nonnegative random variables. So for signed returns with unknown distribution, Cantelli is usually the intended bound.

Moments, Covariance, and MGF

This cluster of cards asks for one of four things: a second moment, a variance, a covariance or correlation, or an mgf value. The key identities are explicit. For second moments use E[X²] = Var(X) + (E[X])². For sums of independent variables add variances. For covariance use Cov(X,Y) = E[XY] - E[X]E[Y]. For mgfs use M_X(t) = E[e^{tX}] and either recognize the family or substitute the given t.

Examples: if H ~ Binomial(10,1/2), then E[H]=5 and Var(H)=10·(1/2)(1/2)=2.5, so E[H²]=2.5+25=27.5. If N₁ and N₂ count 1s and 2s in n fair six-sided dice, then they come from a multinomial split, so Cov(N₁,N₂)=-n(1/6)(1/6)=-n/36. If X ~ Poisson(3), then M_X(t)=exp(3(e^t-1)), so M_X(ln 2)=exp(3(2-1))=e^3.
Precise workflow

For a variance question, first decide whether the pieces are independent. If they are, add variances. If they come from sampling without replacement, expect a finite-population correction. If they are counts from categories that must sum to a fixed total, expect negative covariance between different categories.

For covariance, ask whether the two variables share randomness. If they do, covariance is often the variance of the shared component after centering. If they split a fixed total, covariance is often negative. If they come from disjoint independent pieces, covariance is zero.

For mgfs, either recognize the standard formula directly or plug the requested t into the known mgf. If the prompt asks for mean or variance and gives an mgf, differentiate at zero. If it asks for a plug-in value like M_X(\ln 2), simplify e^{\ln 2}=2 immediately.

Moments and MGF Workflow

To calculate a moment, differentiate the mgf at zero, use a known formula for the distribution family, rewrite the variable as a transform of a simpler one, or use symmetry to show that an odd moment is zero.

Examples: for X ~ N(0,1), odd moments vanish by symmetry and E[X^{2m}] = (2m-1)!!. For a lognormal X = e^Y with Y ~ N(μ,σ²), raw moments come from E[X^r] = E[e^{rY}] = exp(rμ + 0.5r²σ²). For Poisson or binomial, the mgf or factorial-moment route is usually faster than summing the pmf directly.
Step-by-step workflow

Step 1: decide what quantity is being asked for. Mean is E[X], variance is E[X²]-E[X]^2, and a “fourth moment” question usually means the raw moment E[X^4] unless the prompt explicitly says central moment.

Step 2: choose the right route. If an mgf is known and easy, differentiate it at zero. If the distribution is a transform, use the transform directly. For example, if X = e^Y with Y ~ N(μ,σ²), then E[X^r] = E[e^{rY}] = exp(rμ + 0.5r²σ²). That immediately gives E[X] = exp(μ + σ²/2) and E[X²] = exp(2μ + 2σ²), so Var(X) = (e^{σ²}-1)e^{2μ+σ²}.

Step 3: check symmetry or existence. For distributions symmetric about 0, odd raw moments vanish. For heavy-tailed families, verify that the requested moment exists before trying to compute it.

Distribution Relationships

Sums of Bernoulli variables produce binomial counts, rare-event binomials converge to Poisson laws, normalized independent gamma variables produce beta laws, and the quotient of two independent standard normals is Cauchy.

Examples: Binomial(n,p) becomes Poisson(λ) when n is large, p is small, and np = λ; exponential is the continuous analogue of geometric waiting time; the k-th order statistic of uniform draws is beta; and if X and Y are iid standard normal, then X/Y is standard Cauchy.
Exact statements

Binomial → Poisson: if X_n ~ Binomial(n,p_n) with n p_n → λ and p_n → 0, then X_n is approximately Poisson(λ). This is the “many tiny independent chances” regime.

Geometric ↔ Exponential: geometric is the discrete memoryless waiting-time law, exponential is the continuous memoryless waiting-time law. Under a small time-step limit, a geometric waiting time rescales to an exponential one.

Uniform → Beta: if U_{(r)} is the r-th order statistic from n iid Uniform(0,1) draws, then U_{(r)} ~ Beta(r, n-r+1). This is why E[U_{(r)}] = r/(n+1).

Beta ↔ Gamma: if G₁ ~ Gamma(a,θ) and G₂ ~ Gamma(b,θ) are independent with the same scale parameter, then G₁/(G₁+G₂) ~ Beta(a,b).

Gamma ↔ Exponential: Exponential(λ) is the same as Gamma(shape=1, rate=λ). More generally, the sum of k independent Exponential(λ) variables is Gamma(shape=k, rate=λ).

Poisson ↔ Exponential ↔ Gamma: in a Poisson process with rate λ, the count in time t is Poisson(λt), the waiting time to the first arrival is Exponential(λ), and the waiting time to the k-th arrival is Gamma(shape=k, rate=λ).

Birthday Matches

There are two nearby but different birthday expectations. The expected number of matching pairs in a group of n is choose(n, 2) / 365. The expected number of people who have at least one match is n(1 - (364/365)^(n-1)).

Example: with 25 people, the expected number of matching pairs is 300 / 365 ≈ 0.822, while the expected number of people who would raise their hands because someone else shares their birthday is about 1.59.
Why the formulas differ

For matching pairs, put an indicator on each unordered pair of people. Each pair matches with probability 1/365, and there are choose(n, 2) such pairs, so linearity gives the expectation immediately.

For raised hands, fix one person and ask for the chance that at least one of the other n - 1 people shares their birthday. That probability is 1 - (364/365)^(n-1), and summing that same expectation over all people gives the hand count.

If four people share one birthday, that produces 6 matching pairs but only 4 raised hands, so the two quantities are measuring different things.

Symmetry Tricks

Some interview questions look computational until you spot a symmetry. If one player gets one extra fair coin toss after an otherwise symmetric contest, that extra toss is just a fair tiebreaker. If you need three equally likely outcomes from fair coins, use two-flip rounds and restart on one bad pattern.

Examples: getting more heads in 5 fair tosses than someone else gets in 4 fair tosses has probability 1/2. Choosing fairly among three children via HH, HT, TH, and restart-on-TT takes an expected 8/3 flips.
Why these shortcuts work

For the 5-versus-4 coin game, imagine both players first toss 4 coins. The game is completely symmetric up to that point. If they tie, the extra fifth toss is simply a fair tiebreaker for the player with the extra coin.

For the three-child selection trick, two fair flips create four equally likely outcomes. Three outcomes are useful and one means “start over,” so each round succeeds with probability 3/4 and costs 2 flips. That gives expected flips 2 / (3/4) = 8/3.

Circle and Modulo Tricks

Circle problems often hide a semicircle criterion, and divisibility questions with many fair dice often hide modular uniformity. These are both places where one structural observation can replace a long count.

Examples: three random points on a circle contain the center in their triangle with probability 1/4. The sum of 10 fair six-sided dice is divisible by 6 with probability exactly 1/6.
Why the clean answers appear

The center lies inside the triangle exactly when the three chosen points are not all contained in any semicircle. The complement event has probability 3/4, leaving the desired probability 1/4.

For fair six-sided dice, each die is uniform modulo 6, since the faces correspond to the residues 1,2,3,4,5,0 equally often. Adding an independent uniform residue modulo 6 keeps the sum uniform modulo 6, so every residue class occurs with probability 1/6.

Regression and Bayesian Updates

In simple linear regression, the slope is β₁ = ρ σ_Y / σ_X and the intercept is β₀ = ȳ - β₁ x̄. If the variables are standardized, the slope is just the correlation. In that same one-factor world, R² = ρ², the explained variance is ρ² Var(Y), and the residual variance is (1 - ρ²) Var(Y).

Examples: if Corr(X,Y)=0.6, sd(X)=2, and sd(Y)=5, then the slope of Y on X is 0.6 × 5 / 2 = 1.5. If Var(Y)=25, then the residual variance is 25(1 - 0.36)=16.
Why the regression formulas stay simple

Least squares is geometry in disguise. The fitted line chooses coefficients so the residual vector is orthogonal to the span of the regressors. In simple regression that orthogonality condition collapses to the familiar slope formula based on covariance divided by variance, which is why correlation and standard deviations are enough to recover the slope mentally.

Normal-error regression MLE is the same problem because the Gaussian log-likelihood is a constant minus a multiple of the sum of squared residuals. So maximizing likelihood and minimizing squared error are literally the same optimization. Through the origin, that gives beta-hat = sum x_i y_i / sum x_i². When the x_i are centered, the intercept falls out as y-bar.

Laplace-error regression changes only one ingredient: squared residuals become absolute residuals. That is why medians and weighted medians appear. The log-likelihood becomes a constant minus a multiple of sum |residual|, so the best-fit coefficient is whichever value balances total weight on both sides instead of balancing squared deviations.

For the Bayesian cards here, the intended model is explicit regression: β ~ N(m₀, τ²) and y_i = β x_i + ε_i with independent ε_i ~ N(0, σ²). In that one-parameter Gaussian regression, the posterior variance is 1 / (1/τ² + Σx_i²/σ²), and the posterior mean is that variance times m₀/τ² + Σx_i y_i / σ².

So the Bayesian regression cards are really sufficient-statistic cards: once you know Σx_i² and Σx_i y_i, you already know the entire posterior for β. If all x_i = 1, the model reduces to a noisy-mean update, and the posterior mean becomes a precision-weighted average of the prior mean and the sample mean.

That is also where shrinkage comes from. Noisy data means small data precision, so the posterior stays closer to the prior. Precise data means large data precision, so the posterior moves toward the regression estimate. For prediction, regression-to-the-mean is still the mental anchor: if X is z standard deviations above its mean, the fitted Y is only ρz standard deviations above its own mean.

Order Statistics

For Uniform(0,1) draws, the expected r-th smallest value is r/(n+1). That means the expected largest is n/(n+1), the expected second-largest is (n-1)/(n+1), and so on. For fair m-sided dice, a fast mental approximation is the same continuous rule scaled to {1, ..., m}: use ((n-j+1)/(n+1))(m+1) for the j-th largest.

Examples: among 5 uniform draws, the expected second-largest is 4/6 = 2/3. Among 5 rolls of a fair 20-sided die, the expected second-largest is roughly (4/6) × 21 = 14. The minimum of n uniforms has variance n/((n+1)^2(n+2)).
Why these formulas are so structured

For Uniform(0,1), the order statistics carve the unit interval into n + 1 exchangeable spacings. By symmetry, each spacing has expected length 1/(n+1). Summing the first r spacings gives the expected r-th smallest value r/(n+1). That is why the formula is exact and so clean.

For discrete dice, exact expectations come from survival probabilities: the j-th largest roll is at least k exactly when at least j rolls land in {k, k+1, ..., m}. The site computes that exact discrete answer, but the mental shortcut treats the die as a continuous uniform grid, which is why the accepted range covers both.

Exponentials are special because the ordered gaps are explicit. If X_(1) < ... < X_(n) are the sorted values, then X_(1) has mean 1/(nλ), the next gap X_(2)-X_(1) has mean 1/((n-1)λ), and in general the gaps are independent exponentials with rates nλ,(n-1)λ,...,λ. So E[X_(n)] = 1/(nλ) + 1/((n-1)λ) + ... + 1/λ = H_n/λ. More generally, the j-th largest value is X_(n-j+1), so E[X_(n-j+1)] = (1/λ)\sum_{m=j}^n 1/m. For mental math, replace H_n by ln n + γ.

The minimum of n uniforms is Beta(1,n), so E[min] = 1/(n+1) and Var(min) = n/((n+1)^2(n+2)). Broken-stick spacings are the same exchangeability idea in disguise: if you cut [0,1] at random, the m segment lengths form a symmetric Dirichlet vector, so the expected j-th largest spacing is (1/m)∑_{k=j}^m 1/k.

The polygon version of the broken-stick problem is just the “largest piece” test: the pieces can form a polygon exactly when no piece exceeds 1/2, since the total length is 1. And for the random-interval cards, all intervals intersect exactly when the middle cut between the n-th and (n+1)-st ordered endpoints is crossed by every pair, which is why the answer becomes a pure counting ratio 2^n/(2n choose n).

Kelly and Hedge Ratios

Kelly sizing and minimum-variance hedging are both “optimize a quadratic-looking expression” problems, but they optimize different things. Kelly maximizes expected log-growth. Hedge ratios minimize return variance.

Examples: for a bet that wins b dollars with probability p and loses 1 dollar otherwise, the Kelly fraction is (bp-q)/b. If you hedge R_A with R_A - hR_B, the variance-minimizing choice is h^* = Cov(R_A,R_B)/Var(R_B) = ρσ_A/σ_B.
Why these formulas are so short

For Kelly, if you bet fraction f, the one-step log-growth is p ln(1 + bf) + q ln(1 - f). Differentiate and set to zero: pb/(1+bf) - q/(1-f) = 0. Solving gives f^* = (bp-q)/b. In the percentage-return version with upside u and downside d, the same derivative gives f^* = (pu-qd)/(ud).

For hedging, expand Var(R_A - hR_B) = Var(R_A) - 2hCov(R_A,R_B) + h^2 Var(R_B). Differentiate with respect to h and set the derivative to zero. That gives h^* = Cov(R_A,R_B)/Var(R_B). If the prompt gives correlation and volatilities instead, substitute Cov = ρσ_Aσ_B to get h^* = ρσ_A/σ_B.

Estimation and MLE

Most one-parameter MLE problems collapse to one of a few memorized templates: Bernoulli and binomial give sample proportion, Poisson gives sample mean, exponential rate gives reciprocal sample mean, normal mean gives sample mean, uniform upper-endpoint gives sample maximum, and Laplace location gives sample median.

Examples: if 14 of 20 coin flips are heads, p-hat = 14/20 = 0.7. If exponential waiting times average 2.5, then the rate MLE is 1/2.5 = 0.4.
Why the same few summaries keep appearing

MLE is about finding the parameter value that makes the observed sample most plausible. Once you take logs, the pieces that do not depend on the parameter disappear, and the whole problem often collapses to one statistic. For Bernoulli data it is the number of successes, for Poisson data it is the total count, for exponentials it is the total waiting time, and for a uniform endpoint it is the sample maximum because smaller endpoints make the sample impossible.

That is also why medians show up for Laplace models. The Laplace log-likelihood is proportional to minus total absolute deviation, so maximizing it means choosing the point that minimizes absolute deviations. In one dimension, that minimizer is the median.

In the regression versions, the same principle survives with more structure. Normal errors lead to squared-loss minimization, hence least squares. Laplace errors lead to absolute-loss minimization, hence medians or weighted medians after rescaling. The important habit is to ask what summary statistic the log-likelihood is really paying attention to.

Fisher Information

Fisher information is the amount of local curvature in the log-likelihood with respect to the parameter. For one observation, it is I_1(θ) = E[(∂_θ ℓ)^2], or equivalently I_1(θ) = -E[∂_θ² ℓ] when the regularity conditions are valid. For n independent observations, information adds: I_n(θ) = n I_1(θ).

Examples: one Bernoulli(p) observation has information 1 / (p(1-p)), so at p = 1/2 that is 4. For n iid Poisson(λ) observations, total information is n/λ, so the large-sample standard error scale for λ̂ is about √(λ/n). For X_i ∼ N(μ,σ^2) with known σ^2, information is n/σ^2, so the natural scale for estimating μ is σ/√(n).

Workflow and interpretation

Start from one observation. Differentiate the log-likelihood with respect to the parameter, then square and take expectation, or take the negative expected second derivative. For mental math, the second-derivative route is often shorter.

The core one-observation formulas worth knowing are: Bernoulli(p): 1/(p(1-p)), Binomial(m,p): m/(p(1-p)), Poisson(λ): 1/λ, Exponential(rate λ): 1/λ^2, Geometric(p) on {1,2,…}: 1/(p^2(1-p)), N(μ,σ^2) for μ with known variance: 1/σ^2, and N(μ,v) for v=σ^2 with known mean: 1/(2v^2).

Once you have I_n(θ), there are three common outputs. The information itself is I_n(θ). The Cramér–Rao lower bound for an unbiased estimator is 1/I_n(θ). The asymptotic MLE standard-error scale is 1/sqrt(I_n(θ)). Those are different quantities and should not be conflated.

For normal observations with a common mean but unequal known variances, information still adds one piece at a time: if X_i ∼ N(μ,σ_i^2) independently, then I(μ) = ∑ 1/σ_i^2. So precise observations count more than noisy ones, and the efficient estimator is the inverse-variance weighted average.

The interpretation is: larger information means steeper likelihood curvature, which means the data pin down the parameter more tightly. That is why the variance lower bound shrinks as information grows.

Worked example: suppose X_1,…,X_{25} are iid Bernoulli(1/4). One observation has information I_1(p)=1/(p(1-p))=1/((1/4)(3/4))=16/3. So total information is I_{25}(p)=25 · 16/3 = 400/3. The Cramér–Rao lower bound is the reciprocal: Var(p̂) ≥ 1/I_{25}(p)=3/400=0.0075. If the prompt instead asked for the large-sample MLE standard error, take a square root and get √(3/400) ≈ 0.0866.

Cramér–Rao and Rao–Blackwell

Cramér–Rao and Rao–Blackwell answer different questions. Cramér–Rao says: among unbiased estimators, variance cannot go below a certain bound. Rao–Blackwell says: given an unbiased estimator, conditioning it on a sufficient statistic produces another unbiased estimator with variance no larger.

Example: if X_1,…,X_n ∼ Bernoulli(p), then X_1 by itself is an unbiased estimator of p, but the sufficient statistic is T = ∑ X_i. Conditioning gives E[X_1 | T] = T/n = X̄. So Rao–Blackwell turns a noisy one-sample estimator into the sample mean, which has lower variance and in fact hits the Cramér–Rao bound p(1-p)/n.
Why Rao–Blackwell matters

Suppose U is any unbiased estimator of θ and T is a sufficient statistic. Then U^*(T) = E[U | T] has the same expectation as U but variance no larger. The reason is the law of total variance: Var(U) = E[Var(U | T)] + Var(E[U | T]). The first term is nonnegative, so replacing U by E[U | T] can only improve or tie the variance.

Sufficiency means that once you know T, the rest of the sample carries no additional information about the parameter. So any remaining randomness in an estimator beyond T is just noise. Rao–Blackwell removes that noise by averaging over all samples with the same sufficient statistic.

This is the constructive counterpart to Cramér–Rao. The bound tells you what variance is impossible to beat; Rao–Blackwell tells you how to systematically improve a bad unbiased estimator. When the sufficient statistic is also complete, Lehmann–Scheffé says the Rao–Blackwellized estimator is the unique UMVU estimator.

Options and Put-Call Parity

For European options on a non-dividend stock, the core identity is C - P = S - K e^{-rT}. At zero rates this becomes C - P = S - K. Many interview questions are really just this identity in disguise, or a one-step binomial valuation, or the short-dated at-the-money heuristic call ≈ 0.4 S σ sqrt(T).

Examples: if S = 100, K = 95, and a call costs 8 with zero rates, then the put is about 3. If S = K = 100, σ = 40%, and T = 0.25, an at-the-money call is roughly 0.4 × 100 × 0.4 × 0.5 = 8.
Exact workflow

Before doing anything else, ask whether the problem is really a parity problem. If call and put share the same strike and expiry, C - P behaves like a forward, so missing prices, implied spot values, and synthetic positions often become one-line calculations.

Cash-flow discounting is the same idea with no optionality attached. Under a continuously compounded risk-free rate r, K dollars paid at time T is worth K e^{-rT} today, and X dollars today grows to X e^{rT} by time T. For small rT, the quick check is e^{-rT} ≈ 1 - rT and e^{rT} ≈ 1 + rT.

For rough pricing, separate intrinsic value from time value. Near expiry, deep in-the-money calls are close to S - K e^{-rT}, deep out-of-the-money calls are near zero, and short-dated at-the-money options live on the scale S σ sqrt(T) with the constant about 0.4 for a call and about 0.8 for a straddle.

In one-step trees, the fair option price is just the discounted risk-neutral expected payoff. With zero rates and a symmetric up/down tree around today’s price, that often collapses to a plain average of the terminal payoffs.

Digital calls are closely tied to tight call spreads. A narrow call spread divided by its strike width behaves like a digital payoff, which is why a fast market estimate for a 1-dollar digital is often just (C(K) - C(K + h)) / h.

Fixed Income and Present Value

Most fixed-income mental-math questions are present-value questions in disguise. Discount each cash flow, add them up, and use quick anchors like e^{-rT} ≈ 1 - rT for small rT. Coupon bonds are just bundles of discounted coupons plus discounted principal.

Examples: a zero-coupon payment of 100 in 2 years at a continuously compounded 5% rate is worth 100e^{-0.1} ≈ 90.5. A perpetual bond paying 6 each year at yield 4% is worth about 6/0.04 = 150.
Exact workflow

The first split is between continuous-compounding and simple discrete discounting. Under continuous compounding, each payment C_t at time t contributes C_t e^{-rt}. Under annual compounding at yield y, it contributes C_t/(1+y)^t. Once the compounding convention is clear, everything else is just cash-flow bookkeeping.

A coupon bond with face value F, coupon cF, maturity n, and annual yield y is worth ∑_{t=1}^n cF/(1+y)^t + F/(1+y)^n. If the coupon rate equals the yield, the price sits near par. If coupon is above yield, price is above par; if coupon is below yield, price is below par.

An annuity is the same geometric-sum idea with equal cash flows. The level-payment present value is A(1-(1+y)^{-n})/y. A perpetuity or consol is the infinite-horizon limit A/y. Many interview fixed-income questions reduce to recognizing which of those templates the cash-flow stream matches.

Greeks and Hedges

Delta is the first thing to estimate mentally. At the-money call delta is around 0.5, deep in-the-money call delta is near 1, deep out-of-the-money call delta is near 0, and put delta is just call delta minus 1. For fast hedge questions, multiply option delta by contract size and number of contracts.

Examples: 100 call contracts with delta 0.52 and contract size 100 carry delta about 5200, so you short about 5200 shares to hedge. If a call has delta 0.62, the matching put delta is about -0.38.
Why the quick rules work

In Black-Scholes, call delta is N(d₁). So if d₁ is around 0, delta is around 0.5; if d₁ is large positive, delta approaches 1; if large negative, it approaches 0. Put delta follows from parity: Δ_put = Δ_call - 1.

Vega is largest for at-the-money options because that is where changing volatility moves the most probability mass across the strike. A good mental scale is vega ≈ 0.4 S sqrt(T) for a full 100-vol-point move, or multiply by 0.01 for a one-vol-point move.

Gamma and vega are positive for long vanilla options, while theta is usually negative. Those sign rules matter because many hedge questions are really asking whether you are long or short convexity, or whether a portfolio’s delta will rise or fall after a move.

Ito and GBM

For a transformed geometric Brownian motion, begin with: dS/S = μ dt + σ dW. If the transform is a power, write it explicitly as X = S^k. Then the transformed process satisfies dX/X = (kμ + 0.5k(k-1)σ²) dt + kσ dW. If the transform is logarithmic, use d ln S = (μ - 0.5σ²) dt + σ dW.

Examples: if dS/S = 8% dt + 30% dW and X = S², then k=2, so dX/X = (2·8% + 0.5·2·1·30%^2)dt + 2·30%dW. Therefore the proportional drift is 16% + 9% = 25% and the proportional volatility is 60%. If X = ln S, then the drift is 8% - 0.5·30%^2 = 3.5% and the volatility is 30%.
Why the power rule is so convenient

Ito’s lemma says df(S) = f'(S)dS + 0.5 f''(S)(dS)². For GBM, (dS)² = σ² S² dt. If f(S) = S^k, then f'(S) = kS^{k-1} and f''(S) = k(k-1)S^{k-2}, so both terms stay proportional to S^k.

The signed diffusion coefficient in dX/X is kσ. The volatility magnitude is |k|σ. Those are different when k < 0. For example, if X = 1/S, then the diffusion term is -σ dW, but the volatility is still σ.

For X = S^k, use the power-transform formula above. For X = ln S, use its separate formula. Then select the requested drift coefficient or volatility magnitude and substitute the given values.

Stopping and Martingales

One-step stopping problems are threshold problems: compare what you can lock in right now to the continuation value of using the option to keep going. Martingale stopping problems are different: if the process is fair, optional stopping says timing alone does not create expected value.

Examples: with one free die reroll, the continuation value is 3.5, so keep 4, 5, 6. For a symmetric stock that moves up or down by 1 each minute, the expected sale price stays at the current price under any bounded stopping rule.
Exact workflow

If you get exactly one chance to continue, compute the continuation value first and keep only outcomes above it. That turns many interview stopping questions into simple compare-against-a-threshold drills.

If the underlying process is a martingale, separate the question of variance reduction from the question of expected value. Stopping can change the distribution, but under the standard fair bounded setup it does not change the mean.

Classic Puzzles: Noodles

If you have n noodles and randomly pair all loose ends, the expected number of loops is 1 + 1/3 + 1/5 + ... + 1/(2n - 1).

Example: for 4 noodles, the expectation is 1 + 1/3 + 1/5 + 1/7 ≈ 1.676.
Classic Puzzles: Runs

For n fair coin flips, the expected total number of runs is (n + 1) / 2. A good mental rule for the longest run of heads is that it sits around log₂ n.

Example: in 20 flips, the expected number of runs is 10.5. In 64 flips, the longest head run is mentally around 6.
Return to Origin via Stirling

For a simple symmetric random walk, the probability of being back at the origin after 2n steps is choose(2n, n) / 2^(2n). Stirling turns that into the fast mental heuristic 1 / sqrt(pi n).

Example: after 20 steps, a walk returns to the origin with probability choose(20, 10) / 2^20 ≈ 1 / sqrt(10 pi) ≈ 0.178.
Where the approximation comes from

Start from choose(2n, n) = (2n)! / (n!)² and apply Stirling’s formula m! ≈ sqrt(2 pi m) (m/e)^m. The exponential pieces simplify cleanly: (2n/e)^(2n) divided by (n/e)^(2n) leaves 2^(2n) = 4^n.

The square-root terms leave the only surviving denominator: sqrt(4 pi n) / (2 pi n) = 1 / sqrt(pi n). So choose(2n, n) is approximately 4^n / sqrt(pi n). After dividing by 2^(2n) = 4^n, the return probability becomes about 1 / sqrt(pi n).

First Collision Time

If you keep drawing uniformly from {1, ..., n} with replacement, the expected time until the first repeated value is about sqrt(pi n / 2). The cruder but very useful scale heuristic is that collisions start to appear when k² / (2n) gets near 1, so the answer should live on the order of sqrt(n).

Example: for n = 365, the estimate is sqrt(pi × 365 / 2) ≈ 23.9, which is the usual birthday-paradox scale.
Why the square-root law appears

The probability of no collision in the first k draws is (1)(1 - 1/n)(1 - 2/n)...(1 - (k-1)/n). When k is much smaller than n, take logs and use ln(1 - u) ≈ -u to get P(no collision by k) ≈ exp(-k(k-1)/(2n)).

That shows the transition from “unlikely collision” to “likely collision” happens when k(k-1)/(2n) is around 1, which already gives the right mental scale k ≈ sqrt(2n). To get the expectation, sum the tail probabilities or approximate by an integral: E[T] ≈ ∫₀^∞ exp(-x²/(2n)) dx = sqrt(pi n / 2).

Classic Puzzles: Waiting for a Streak

For any fair-coin pattern w, the expected waiting time until you first see it is sum 2^k over all lengths k for which the first k symbols of w equal the last k symbols of w. This is the standard martingale / betting-system result. Runs of heads are just the special case where every prefix is also a suffix, so HHH gives 2 + 4 + 8 = 14.

Examples: for HTH, the matching prefix/suffix lengths are 1 and 3, so the mean wait is 2 + 8 = 10. For HTHT, the matching lengths are 2 and 4, so the mean wait is 4 + 16 = 20.
Why overlap lengths appear

The clean proof starts a new bettor on every flip. Each bettor bets sequentially on the letters of the target pattern and doubles their stake whenever they survive one more step. With fair-coin payouts, the total wealth process is a martingale.

At the stopping time when the pattern first appears, exactly the bettors whose starting positions align with a prefix-suffix overlap are still alive. Their payouts are 2^k for the overlap lengths k, so optional stopping turns the expected waiting time into the sum of those powers of 2.

Classic Puzzles: Permutations and Coupons

Several classics live here. The expected number of cycles in a random permutation of n items is H_n, the expected number of fixed points is 1, the derangement probability is \sum_{k=0}^n (-1)^k/k! and is close to 1/e for moderate n, and the expected time to collect n equally likely coupon types is n H_n.

Example: for 4 coupon types, the expected collection time is 4(1 + 1/2 + 1/3 + 1/4) ≈ 8.33.
Classic Puzzles: Coupon Collector Variance and Intervals

Write the full collection time as a sum of stage times. Once you have already seen n - i coupon types, the chance that the next draw gives a new type is i / n. So the time spent in that stage is geometric with mean n / i and variance (1 - i/n) / (i/n)^2 = n²/i² - n/i.

Adding the stage variances gives Var(T_n) = n² sum(1/i²) - n H_n. For mental math at moderate or large n, a fast shortcut is Var(T_n) ≈ (pi²/6)n² - n(ln n + 1/2) ≈ 1.65 n² - n(ln n + 1/2). The first term is the main scale: the standard deviation is on the order of 1.28n.

Example: for n = 20, the mean is 20H_20 ≈ 71.95. The exact variance is about 566.51, so the standard deviation is about 23.80. A rough central 95% interval from the mean-variance heuristic is 71.95 ± 1.96 × 23.80 ≈ [25.3, 118.6].
Where the variance formula comes from

Let X_i be the number of draws needed to go from n-i seen coupon types to n-i+1 seen coupon types. Then X_i is geometric with success probability p_i = i/n, and the whole stopping time is T_n = X_n + X_{n-1} + ... + X_1.

Because each stage starts fresh after the previous success, those geometric stage times are independent. Therefore Var(T_n) = sum Var(X_i). For a geometric random variable on {1,2,3,...}, the variance is (1-p)/p², so Var(X_i) = (1 - i/n)/(i/n)² = n²/i² - n/i. Summing over i = 1, ..., n gives n² sum(1/i²) - n sum(1/i) = n² sum(1/i²) - nH_n.

How to use it for intervals

The coupon-collector distribution is skewed, so a normal interval is only a rough heuristic, not an exact theorem. But as a fast mental summary, it is often good enough to use T_n ≈ mean ± z × sd with mean nH_n and standard deviation sqrt(Var(T_n)).

So a one-sigma band uses z = 1, and a rough central 95% band uses z ≈ 1.96. In practice, many interview-style estimates round that to about 2.
